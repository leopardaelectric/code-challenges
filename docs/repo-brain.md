# repo-brain — local knowledge layer

A self-updating [gbrain](https://github.com/garrytan/gbrain) knowledge layer that
lives inside this repo, indexes **code + markdown notes**, stays current
**automatically on every change**, and runs **fully locally** (PGLite + on-device
Ollama embeddings — no server, no cloud, no API key). AI agents query it as the
repo's knowledge layer via MCP.

## What's committed

```
.brain/                  # COMMITTED  markdown notes that compound (source of truth)
├── decisions/           #            architectural / product decisions
├── context/             #            durable how-it-works / gotchas / glossary
└── learnings/           #            postmortems, debugging notes
.githooks/               # COMMITTED  post-commit/merge/checkout/rewrite + setup.sh + watcher
AGENTS.md / CLAUDE.md    # COMMITTED  brain-first instructions for agents
.gbrain/                 # GITIGNORED local PGLite index + config (regenerated by setup.sh)
```

The markdown in `.brain/` is committed and shared; the derived `.gbrain/` index
is per-machine and gitignored — it rebuilds locally from `setup.sh`.

> **Single default source.** Both code and notes are indexed into the brain's
> **default** source, so `gbrain query` / `gbrain serve` find everything without
> per-source routing. There is intentionally **no `.gbrain-source` pin** — a pin
> to a named source breaks this (`gbrain import` always writes to the default
> source, and queries pinned to a missing source error out with "Source not
> found"). The MCP server name is derived from the repo basename.

## Prerequisites

- **gbrain** on your PATH (v0.42+):
  ```bash
  git clone https://github.com/garrytan/gbrain.git && cd gbrain && bun install && bun link
  ```
- **Ollama** for fully-local, offline embeddings — no API key, no network egress.
  **Install the official app, NOT the Homebrew formula:**
  ```bash
  brew install --cask ollama      # or download from https://ollama.com
  open -a Ollama                  # starts the daemon at http://localhost:11434
  ```
  > ⚠️ **`brew install ollama` (the formula) ships no inference runner.** Its
  > daemon answers `/api/tags` and accepts model pulls, but **500s on
  > embeddings** with `llama-server binary not found` — the formula bundles only
  > an MLX runner, not the `llama-server` (llama.cpp/GGUF) runner that
  > `nomic-embed-text` needs. The official app/cask bundles the runner. If you
  > already installed the formula: `brew uninstall ollama; brew install --cask ollama`.

  `setup.sh` defaults to `ollama:nomic-embed-text` (768 dims), pulls the model if
  missing, and preflights a real embedding call so this failure is caught early.
  Override with `BRAIN_EMBED_MODEL` / `BRAIN_EMBED_DIMS`.

## One-time setup (per machine)

```bash
./.githooks/setup.sh
```

Inits the local PGLite brain at `./.gbrain`, indexes the code (`gbrain sync
--strategy code`) and the committed notes (`gbrain import .brain`) into the
default source, activates the hooks (`git config core.hooksPath .githooks`), and
embeds. Prints the MCP wiring command at the end.

## How "always updated" works

| Trigger | Mechanism | Latency |
|---|---|---|
| `git commit` / `merge` / `pull` / `checkout` / `rebase` | git hooks → `_refresh.sh` (backgrounded) → `gbrain sync --strategy code` + `gbrain import .brain/` + `gbrain embed --stale` | seconds after the action |
| uncommitted `.brain/` note edits | `.githooks/brain-watch.sh` (same sequence, 60s poll) | ≤ 60s |
| uncommitted *code* edits | not indexed — the code sync is commit-to-commit; commit to refresh | next commit |

Both are idempotent (gbrain dedups by content hash) and write to the default
source. Code uses the sync bookmark; notes use content-hash import (no
bookmark) — this pair never deletes each other's pages. Running together is
safe. **Never run a plain `gbrain sync` against this repo**: its incremental
pass deletes code pages as "un-syncable" and advances the bookmark, so the
code sync can't re-import them (gbrain 0.42.x).

## Wire your agent to it

```bash
# GBRAIN_HOME is the parent dir — the brain lives at $GBRAIN_HOME/.gbrain.
# Pass it via --env so it persists in the MCP server config (a command-prefix
# env var is NOT captured by `claude mcp add`).
claude mcp add brain_$(basename "$(pwd)" | tr '[:upper:] ' '[:lower:]-' | tr -cd 'a-z0-9-') --env GBRAIN_HOME="$(pwd)" -- gbrain serve
```

`AGENTS.md` / `CLAUDE.md` instruct agents to do **brain-first lookup**
(`query` → `search` → code tools) before external calls, and to write durable
findings back into `.brain/` so knowledge compounds.

## Verify

```bash
# Run from the repo root. GBRAIN_HOME must be an ABSOLUTE path (relative paths
# like "." don't resolve); the brain lives at $GBRAIN_HOME/.gbrain.
GBRAIN_HOME="$(pwd)" gbrain stats
GBRAIN_HOME="$(pwd)" gbrain query "what is this repo"
GBRAIN_HOME="$(pwd)" gbrain search "<text you just added>"
tail -f .gbrain/refresh.log
```

## Caveats / tuning

- **Fully local / offline.** PGLite, MCP, and embeddings (Ollama) run on your
  machine. Ollama must be running for sync/index to embed; if no embedding
  backend is reachable, gbrain degrades to keyword-only search rather than failing.
- **Build output gets indexed.** `gbrain sync --strategy code` walks the
  filesystem and does **not** honor `.gitignore`; it only prunes `node_modules`,
  `.raw`, `ops`, and dot-dirs. A fresh clone has no `dist/`/`build/`, but if your
  checkout is built, those get indexed as noise. Mitigate by `rm -rf dist build`
  before setup, or delete the pages afterward (`gbrain delete <slug>`).
- **Local-embedding quality.** `nomic-embed-text` (768d) is lower-dimensional
  than cloud code-tuned models, so semantic recall may be a bit lower. Switch
  later with `gbrain reinit-pglite --embedding-model <m> --embedding-dimensions <n>`
  (PGLite can't resize the vector column in place — wipe-and-reindex).
- **Big repos:** a full code re-sync on every commit can be slow. If so, drop the
  code step from `.githooks/_refresh.sh` and let the watcher / a periodic job
  handle code.
- `.brain/` meta filenames `README.md`, `index.md`, `schema.md`, `log.md` are NOT
  indexed by gbrain sync — put real content elsewhere.
